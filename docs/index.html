🏛️

Clean Architecture

Four layers — Domain, Application, Infrastructure, API — with strict inward dependency flow. Swap SQL Server for Cosmos DB without touching business logic.

🗃️

Repository + Unit of Work

IExpenseRepository and IUnitOfWork interfaces in Domain; EF Core implementations in Infrastructure. Fully mockable for unit tests.

🛡️

RFC 7807 Problem Details

Single global ExceptionHandlingMiddleware converts NotFoundException → 404, ValidationException → 422. No stack traces in production responses.

🔐

Managed Identity + Key Vault

Zero secrets in config or source control. Web App uses system-assigned identity to fetch the connection string from Key Vault at runtime.

📦

Azure Bicep IaC

Entire infrastructure defined as code — App Service, Azure SQL, Key Vault, RBAC role assignments. One script to deploy everything from scratch.

🔄

GitHub Actions CI/CD

Push to main → build → test → deploy. Uses Azure publish profile secret. Zero-downtime deployment via App Service slot swap pattern.

📱

Power Apps Canvas App

3-screen Canvas App with Custom Connector. Full Power Fx formulas for loading, saving, deleting expenses and a summary chart by category.

🧪

Unit Tests (xUnit + Moq)

Service layer and validators fully covered. FluentAssertions for readable assertions. Repository mocked — no real database needed to run tests.

⚡

Cancellation Tokens

Every async method propagates CancellationToken from controller all the way to EF Core queries. Graceful cancellation on client disconnect.

5 Interview Talking Points
Pre-loaded answers for "walk me through your architecture".

1. Clean Architecture Dependency Flow

"Domain has zero external dependencies. API never touches EF Core directly — it only calls Application interfaces. I can swap SQL Server for Cosmos DB by changing only the Infrastructure project."

2. Repository + Unit of Work Testability

"Interfaces live in Domain, implementations in Infrastructure. The service layer calls SaveChangesAsync() once — single commit boundary. Unit tests mock IUnitOfWork without touching a real database."

3. RFC 7807 Problem Details

"One middleware catches all exceptions and maps them to structured Problem Details responses. NotFoundException → 404, ValidationException → 422. No stack traces leak to production."

4. Managed Identity + Key Vault (zero secrets)

"The Web App gets a system-assigned identity, which is granted Key Vault Secrets User via Bicep RBAC. The connection string reference in App Settings — raw secret never exists anywhere."

5. Cancellation Tokens Throughout

"Every async method propagates CancellationToken from controller to EF Core. When a client disconnects, the token fires, EF abandons the in-flight query, and the thread is freed immediately."
